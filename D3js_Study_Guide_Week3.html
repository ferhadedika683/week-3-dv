1. What is D3.js?

🔑 Core Definition D3 stands for Data-Driven Documents. It is a JavaScript library that lets you bind data to DOM elements and transform those elements based on your data — enabling rich, interactive, animated visualizations directly in the browser.

Easy Explanation

Think of D3 as a translator. You give it a table of numbers, and it turns them into shapes, bars, maps, or graphs on a web page — and those visuals react to your data changes automatically.

Who Created It?

Mike Bostock created D3.js to take full advantage of modern web standards (CSS3, HTML5, SVG) instead of relying on proprietary plugins like Flash.

Why D3 was Needed

Before D3, web visualizations required proprietary tools. D3 fills the need for web-accessible, sophisticated data visualization using open standards.

Key Capabilities

  • Create rich interactive and animated content based on data
  • Tie visual content to existing web page elements
  • Expose full power of CSS3, HTML5, and SVG
  • Avoid proprietary representation — open, standards-based
  • Afford extraordinary flexibility in chart design

How D3.js Works — The Core Concepts

D3's workflow revolves around two key ideas:

Selection

A selection is the group of data and elements taken together. You select HTML/SVG elements you want to work with.

d3.select("body")
d3.selectAll("circle")

Binding

A binding is the association of a dataset with selected elements. Once bound, element appearance is derived from the data.

.data([10, 20, 30])
.enter().append("rect")
📌 Exam Tip D3 is about selecting elements and binding data to them. The appearance of elements is derived from the bound data. This is the fundamental philosophy of D3.

2. Web Standards

Core Idea Before using D3, you must understand the five web standards it heavily relies on. These form the foundation of every D3 visualization.

1. HTML

HyperText Markup Language

Used to structure the content of the web page. Defines what elements exist (headings, divs, paragraphs, etc.).

2. DOM

Document Object Model

Every HTML tag gets converted to a DOM element with a hierarchy (parent-child tree). D3 manipulates this tree.

3. CSS

Cascading Style Sheets

Styles a web page, making it pleasant and interactive in appearance. D3 applies CSS styles dynamically.

4. SVG

Scalable Vector Graphics

A way to create images using text/code. D3 primarily draws charts using SVG elements like circles, rectangles, paths.

5. JavaScript

JS interacts with HTML/DOM elements to make the web UI interactive. D3 is a JavaScript library.

The DOM Hierarchy — Explained

From the slide's "Listing 1.1" example:

<!doctype html>
<html>
  <head>
    <script src="d3.v4.min.js"></script>   ← child of <html>
  </head>
  <body>
    <div id="someDiv" style="width:200px;height:100px;border:black 1px solid;">
                                                ← child of <body>
      <input id="someCheckbox" type="checkbox" />
                                                ← child of <div>
    </div>
  </body>
</html>

The hierarchy is: html → head / body → div → input. D3 navigates this tree to select and modify elements.

Why Web Standards Matter D3 deliberately builds on web standards so that visualizations work in any modern browser without plugins. The quote from Mike Bostock: D3 "avoids proprietary representation and affords extraordinary flexibility, exposing the full capabilities of web standards such as CSS3, HTML5, and SVG."

Selecting DOM Elements with D3

D3 allows you to manipulate DOM elements. First, you must select them:

MethodDescription
d3.select(css-selector)Returns the first matching element in the HTML document based on the specified CSS selector
d3.selectAll(css-selector)Returns all matching elements in the HTML document based on the specified CSS selector

Three Ways D3 Can Select Elements

By Element

d3.select("body")
d3.selectAll("div")
d3.selectAll("p")

By ID

d3.select("#chart")
d3.select("#someDiv")

Uses # prefix (CSS ID selector)

By CSS Class

d3.selectAll(".bar")
d3.select(".container")

Uses . prefix (CSS class selector)

Hello World — First D3 App

From the lecture slides — the very first D3 program:

<head>
  <title>TEB3133 Data Visualization</title>
  <script src="https://d3js.org/d3.v7.min.js"></script>
</head>
<body>
  <div id='chart'> </div>
  <script>
    d3.select("body").append("chart")
      .style("border", "1px black solid")
      .html("hello world");

    d3.select("chart")
      .style("background-color", "pink")
      .style("font-size", "24px")
  </script>
</body>

What this code does

  • Selects body with d3.select()
  • Appends a "chart" element to the body
  • Applies inline styles with .style()
  • Sets inner HTML with .html()
  • Then selects and styles "chart" separately

Expected Output

A pink bordered box on the page containing:

hello world
⚠ Common Mistake The .style() method can be chained — D3 uses method chaining (each method returns the selection, so you can keep calling more methods on the same line).
⚡ Quick Recall
Setup = D3 Library + Web Server + Editor + Browser
Select one: d3.select() | Select many: d3.selectAll()
Selectors: element tag | #id | .class
Common Mistakes — Setup & DOM Selection
  • Using d3.select() when you want all elements — it only returns the first match
  • Forgetting the # prefix when selecting by ID
  • Forgetting the . prefix when selecting by class
  • Not including the D3 library script tag before trying to use D3 functions
  • Running D3 code before the DOM is loaded (put scripts at end of body or use window.onload)

4. Making Charts Interactive

Overview Static charts show a snapshot. Interactive charts allow users to explore, filter, zoom, and inspect data — transforming passive viewers into active explorers.

Why Interactive Charts?

  • Filter data to specific needs
  • Zoom into regions of interest
  • View details via tooltips
  • Update without page refresh
  • Retrieve real-time data

Popular Libraries

  • Chart.js
  • D3.js
  • Highcharts
  • ApexCharts

4.1 Selecting Chart Content

Definition Chart content selection lets users decide which data to display, instead of showing everything at once.

Sales Dashboard

Select specific products to compare

Weather App

Choose different cities

Stock Market

Compare multiple companies

Why Content Selection is Important

BenefitExplanation
Improves UXUsers focus on relevant information
Reduces Visual ClutterToo much info makes charts hard to read
Supports Comparative AnalysisCompare different datasets easily
Enhances PerformanceLoading only selected data reduces rendering complexity

Methods for Selecting Chart Content

1 Dropdown Menus — Users choose one category 
<select id="dataset">
  <option>Sales</option>
  <option>Profit</option>
  <option>Revenue</option>
</select>

When a user selects a new option, the chart updates accordingly.

2 Checkboxes — Display multiple datasets simultaneously 
<input type="checkbox"> Product A
<input type="checkbox"> Product B
<input type="checkbox"> Product C

Useful for comparative analysis — see multiple series at once.

3 Date Range Filters — Select a time period
  • Last 7 days
  • Last month
  • Last year

Commonly used in financial and business intelligence applications.

4 Interactive Legends — Click to show/hide datasets

Many charting libraries allow users to click on chart legends to toggle datasets. Benefits: cleaner visuals, better comparison, greater user control.

4.2 Zooming In on Charts

Definition Chart zooming allows users to enlarge a specific portion of a chart. Particularly useful for large datasets, time-series data, scientific measurements, and financial data.

Benefits of Zooming

  • Better data exploration — inspect small details
  • Improved accuracy — see small fluctuations
  • Enhanced decision-making — focus without distractions

Challenges of Zooming

  • Performance issues — large datasets slow rendering
  • Context loss — users lose the big picture
  • Mobile compatibility — touch needs extra dev work

Types of Zooming

TypeHow it WorksAdvantage
Mouse Wheel ZoomScroll mouse wheel to zoom in/outFast, intuitive, easy to implement
Drag-to-ZoomDrag a rectangular selection over chart areaPrecise selection, good for large datasets
Pinch-to-ZoomTwo-finger gesture on touch devicesMobile-friendly, familiar smartphone interaction

Zoom Reset Functionality

After zooming, users must be able to return to the original view. Common approaches:

  • Reset Zoom button: <button>Reset Zoom</button>
  • Double-click reset
  • Keyboard shortcut

4.3 Tracking Data Values

Definition Data value tracking allows users to inspect exact values in a chart, instead of estimating visually. Significantly improves usability and accuracy.

Why Track Values?

  • Increased precision — exact values not approximations
  • Better UX — interactive feedback keeps users engaged
  • Enhanced analysis — identify trends and anomalies

4 Common Data Tracking Techniques

1. Tooltips

Small info boxes appearing on hover over a chart element. The most widely used interaction feature.

Month: January
Sales: RM 15,000

2. Crosshair Tracking

Vertical and horizontal lines follow cursor position. Easier data point ID. Used in financial trading platforms for time-series charts.

3. Highlighting Data Points

On hover, a point becomes visually distinct:

  • Larger marker size
  • Different color
  • Increased opacity

4. Data Labels

Values displayed directly on the chart.

Advantages: Immediate visibility, no hovering needed.

Disadvantages: May clutter charts with large datasets.

Example: Q1: 120 · Q2: 150 · Q3: 200

Real-World Tracking Applications

📊 Business Dashboards — Tracking sales and revenue

🏥 Healthcare — Monitoring patient statistics

🌿 Environmental — Observing temperature over time

5. Retrieving Data Using AJAX

🔑 Definition AJAX = Asynchronous JavaScript and XML. A technique that allows web apps to communicate with servers without refreshing the entire page. Modern implementations use JSON rather than XML.

Why Use AJAX with Charts?

Traditional Websites

Require a full page refresh to show updated data. Slow and disruptive to the user experience.

AJAX-Powered Sites

  • Fetch data in the background
  • Update chart contents dynamically
  • Improve responsiveness
  • Enhance user experience

AJAX Workflow — Step by Step

① User Action
→
② JS Sends Request
→
③ Server Processes
→
④ Server Returns Data
→
⑤ Chart Auto-Updates

This entire process occurs without reloading the webpage.

AJAX Request Example — Fetch API

fetch('salesData.json')
  .then(response => response.json())
  .then(data => {
    console.log(data);
  });
Note The Fetch API is the preferred modern approach for asynchronous requests in web applications (replacing older XMLHttpRequest).

JSON Data Example

{
  "months": ["Jan", "Feb", "Mar"],
  "sales": [100, 150, 200]
}

This JSON data can be used directly by charting libraries to update charts dynamically.

Dynamic Chart Updating

Suppose new sales data arrives every minute. AJAX can:

Request Updated Data
→
Modify Chart Dataset
→
Redraw Chart Auto

Users always see current info — without refreshing their browser.

Real-Time AJAX Applications

💰 Financial Dashboards — Stock prices update continuously

🌤 Weather Monitoring — Temperature refreshes regularly

🔌 IoT Systems — Sensor data streams in real time

🛒 E-Commerce — Live sales and inventory updates

AJAX: Advantages vs. Challenges

✓ Advantages

  • Faster UX — only necessary data transferred
  • Reduced server load — no full-page reloads
  • Real-time data access — always current info
  • Better interactivity — charts more responsive

⚠ Challenges

  • Network delays — slow internet affects updates
  • Error handling — must handle failed requests
  • Security — server-side validation essential
  • Data consistency — frequent updates cause sync issues

Maintain AccessibilityKeyboard navigation and screen-reader support
Test with Real UsersGather feedback to improve usability

Workflow: User action → JS request → Server processes → Data returned → Chart updates
Fetch API = modern AJAX method
Common Mistakes — AJAX
  • Thinking AJAX uses XML today — modern AJAX uses JSON
  • Forgetting error handling — network requests can and do fail
  • Not validating data on the server side — security risk
  • Polling too frequently — hammers the server and causes sync issues
  • Confusing synchronous and asynchronous — AJAX is asynchronous (non-blocking)

Complete Cheat Sheet

D3.js Fundamentals

  • D3 = Data-Driven Documents
  • Creator: Mike Bostock
  • Purpose: Bind data to DOM, derive appearance from data
  • Core: Selection + Binding
  • Selection = group of data + elements
  • Binding = associating dataset with elements
  • Standards: CSS3, HTML5, SVG (avoids proprietary)
  • Can create: Charts, Networks, Maps, Dashboards
  • Interactivity: Mouse events, Brush, Zoom, Optimization

Web Standards (5)

  • HTML → structures web page content
  • DOM → tree of HTML elements (hierarchy)
  • CSS → styles pages, applied dynamically by D3
  • SVG → vector images via code (D3's drawing canvas)
  • JavaScript → interacts with DOM, enables interactivity
  • DOM example: html → body → div → input

D3 Setup & Selection

  • Setup: D3 Library + Web Server + Editor + Browser
  • Include: <script src="d3js.org/d3.v7.min.js">
  • d3.select() → FIRST matching element
  • d3.selectAll() → ALL matching elements
  • Select by element: d3.select("body")
  • Select by ID: d3.select("#chart")
  • Select by class: d3.selectAll(".bar")
  • Method chaining: .style().html().attr()

Interactive Charts

  • Content Selection: dropdown, checkbox, date filter, legend
  • Benefits: better UX, less clutter, comparison, performance
  • Zoom types: Mouse Wheel, Drag-to-Zoom, Pinch-to-Zoom
  • Always provide zoom reset (button/double-click)
  • Tracking: Tooltips (most common!), Crosshair, Highlight, Labels
  • Tooltips appear on hover, show exact values
  • Crosshair: used in financial/time-series charts
  • Data Labels: visible always but can clutter

AJAX

  • AJAX = Asynchronous JavaScript and XML
  • Modern AJAX uses JSON not XML
  • Workflow: Action → Request → Process → Data → Update
  • No page reload needed
  • Fetch API: fetch(url).then().then()
  • Advantages: faster, less server load, real-time
  • Challenges: network delays, errors, security, sync
  • Real-time uses: stocks, weather, IoT, e-commerce

Best Practices

  • Keep interactions simple
  • Provide visual feedback (loading states)
  • Optimize rendering performance
  • Ensure mobile/touch compatibility
  • Maintain accessibility (keyboard nav)
  • Test with real users
  • Always handle AJAX errors gracefully
  • Validate data server-side (security!)
